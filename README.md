# MyReading 📚

노션(Notion) 블로그용 이미지 및 에셋 아카이브 저장소입니다.

- 블로그: [블로그 링크](https://frenz104.vercel.app/)
- 노션: [노션 링크](https://frenz104.notion.site/2f9dacf1039981ea9e6dc771bb5aa7e1)

## 📁 폴더 구조

이 저장소는 연도별로 이미지와 에셋을 체계적으로 정리합니다.

```
MyReading/
└── 2026/
    ├── images/     # 2026년 이미지 파일
    └── assets/     # 2026년 기타 에셋 (아이콘, 로고 등)
```

> 💡 필요에 따라 새로운 연도 폴더를 추가할 수 있습니다 (예: 2027/, 2028/ 등).

## 🎯 사용 목적

- 노션 블로그에 사용될 이미지 호스팅
- 블로그 포스트의 스크린샷 및 다이어그램 저장
- 아이콘, 로고 등의 재사용 가능한 에셋 보관
- 체계적인 연도별 파일 관리

## 📝 사용 방법

### 1. 파일 업로드

해당 연도의 적절한 폴더에 파일을 업로드합니다:
- **이미지**: `YYYY/images/` 폴더에 저장
- **기타 에셋**: `YYYY/assets/` 폴더에 저장

### 2. 노션에서 이미지 사용하기

GitHub에 업로드된 파일은 다음과 같은 URL 형식으로 접근할 수 있습니다:

```
https://raw.githubusercontent.com/juyongSong/MyReading/main/YYYY/images/파일명.png
```

또는 (브랜치가 다른 경우):

```
https://raw.githubusercontent.com/juyongSong/MyReading/브랜치명/YYYY/images/파일명.png
```

### 3. 파일 네이밍 규칙 (권장)

가독성과 관리를 위해 다음 규칙을 권장합니다:

- **날짜 포함**: `2026-01-31-screenshot.png`
- **설명적 이름**: `react-component-diagram.png`
- **공백 대신 하이픈 사용**: `my-blog-post-image.png`
- **소문자 사용**: `example-image.png` (대신 `Example-Image.png`)

## 📌 파일 형식

### 이미지 (images/)
- PNG, JPG, JPEG, GIF, SVG
- 스크린샷, 다이어그램, 사진 등

### 에셋 (assets/)
- 아이콘 (ICO, SVG)
- 로고 및 브랜딩 자료
- PDF 문서
- 기타 재사용 가능한 파일

## 💡 팁

1. **이미지 최적화**: 업로드 전에 이미지를 최적화하여 로딩 속도를 개선하세요
2. **일관된 네이밍**: 파일명을 일관되게 유지하면 나중에 찾기 쉽습니다
3. **백업**: 중요한 파일은 별도로 백업하는 것을 권장합니다
4. **커밋 메시지**: 의미 있는 커밋 메시지로 변경 이력을 관리하세요
